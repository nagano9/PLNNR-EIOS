# Information Classification Scheme

## Tujuan

Menetapkan klasifikasi informasi untuk EIOS.

## Levels

- Public
- Internal
- Confidential
- Restricted

## Minimum Controls

| Level | Control |
|---|---|
| Public | Dapat dibagikan |
| Internal | Hanya pengguna internal |
| Confidential | Need-to-know, limited sharing |
| Restricted | Explicit authorization, enhanced logging |

## Pemetaan Contoh

| Level | Contoh artefak PLN NR |
|---|---|
| Public | Framework generik, siaran pers |
| Internal | Standar, pedoman, template |
| Confidential | Kontrak EPC/PPA, model finansial, minutes Direksi |
| Restricted | Data pribadi pegawai, harga & term JV, data due diligence |

## Kepatuhan

Level Restricted untuk data pribadi mengikuti **UU No. 27/2022 (PDP)**; kontrol keamanan mengikuti **ISO/IEC 27001**. Klasifikasi wajib diisi pada setiap knowledge object.
