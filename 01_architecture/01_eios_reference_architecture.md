# EIOS Reference Architecture

## Tujuan

Menjelaskan lapisan arsitektur EIOS dan hubungan antarkomponen.

## Lapisan

1. **Source Layer** — SharePoint, Excel, ERP, email, kontrak, notulen.
2. **Ingestion Layer** — extraction, classification, OCR bila diperlukan.
3. **Knowledge Layer** — knowledge objects, metadata, ontology.
4. **Intelligence Layer** — retrieval, reasoning, comparison, pattern detection.
5. **Decision Layer** — recommendation, alternatives, risks, assumptions.
6. **Experience Layer** — Copilot, ChatGPT, Claude, dashboards, agents.
7. **Governance Layer** — security, audit, quality, lifecycle, model risk.

## Arsitektur Prinsip

EIOS tidak menggantikan system of record. EIOS menjadi intelligence layer di atas system of record.
