# Non-Functional Requirements

## Tujuan

Menetapkan kebutuhan kualitas arsitektur EIOS.

## Requirements

| Area | Requirement |
|---|---|
| Availability | Layanan kritis mengikuti SLA korporat |
| Security | Role-based access, encryption, audit trail |
| Traceability | Jawaban AI menunjuk sumber |
| Performance | Retrieval relevan dalam waktu yang dapat diterima |
| Scalability | Mendukung pertumbuhan knowledge object |
| Interoperability | API dan format terbuka |
| Maintainability | Modular dan version-controlled |
| Portability | Tidak terkunci pada satu model AI |
| Compliance | Selaras GCG BUMN, **UU 27/2022 PDP**, **ISO/IEC 27001** |
| Auditability | Log keputusan & output AI dapat diperiksa SPI |

## Catatan

Tingkat SLA, RPO/RTO, dan enkripsi mengikuti kebijakan TI korporat PLN/Holding. Persyaratan traceability bersifat wajib untuk output AI yang mendukung keputusan material.
