# Enterprise Pattern Library Standard

## Tujuan

Menetapkan format penyimpanan pola berulang.

## Pattern Structure

Name, problem, context, forces, solution, trade-offs, warning signs, examples, related patterns.

## Pattern Domains

Finance, investment, governance, contract, negotiation, risk, project execution, stakeholder management.

## Lesson vs Pattern (kapan naik kelas)

Satu **lesson** = pembelajaran dari satu kejadian. Bila pembelajaran serupa **berulang ≥2–3 kali**, ia dikodifikasi menjadi **pattern** — pengetahuan yang dapat langsung diterapkan (reusable), bukan sekadar catatan historis. Pattern adalah aset bernilai tertinggi pada intelligence balance sheet.

## Kriteria Pattern Layak Publikasi

1. Problem & context jelas dan berulang.
2. Solusi teruji (punya ≥1 contoh nyata/ilustratif).
3. Trade-offs & warning signs eksplisit (kapan TIDAK dipakai).
4. Tertaut ke lesson/decision sumbernya.
