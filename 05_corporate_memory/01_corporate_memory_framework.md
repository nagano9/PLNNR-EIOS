# Corporate Memory Framework

## Tujuan

Menetapkan cara organisasi menyimpan konteks dan pengalaman agar tidak hilang saat terjadi rotasi personel.

## Memory Layers

- Institutional facts
- Decision history
- Assumptions
- Negotiation history
- Project lessons
- Stakeholder context
- Policy rationale
- Patterns

## Principle

Corporate memory harus menyimpan bukan hanya apa yang diputuskan, tetapi juga mengapa.

## Pemicu Penangkapan (Capture Triggers)

Konteks wajib ditangkap pada momen: financial close, penandatanganan kontrak (PPA/EPC/JV), keputusan Direksi/Dekom, negosiasi mitra strategis, perubahan asumsi material, dan post-mortem proyek. Momen rotasi/mutasi personel kunci memicu knowledge handover terstruktur.

## Standar Rujukan

Kerangka ini mengoperasionalkan **ISO 30401** dan mendukung akuntabilitas GCG — rasional keputusan tetap tertelusuri meski personel berganti.
